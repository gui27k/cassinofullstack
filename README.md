# 🎰 xdogcassino — Hub de Mini-Games & Simulador de Probabilidade

## 📋 Objetivo do Projeto
O **xdogcassino** é um protótipo de plataforma de entretenimento e simulação visual focado no desenvolvimento de uma interface moderna (*Cyber-Dark Mode*). O objetivo principal é estudar a arquitetura Full-Stack, persistência de sessões no navegador através de cookies/localStorage e a integração de mecânicas dinâmicas de cliques baseadas em algoritmos matemáticos de probabilidade. 

> ⚠️ **NOTA DE DISCLAIMER:** Este projeto é estritamente educacional, simulado e voltado para portfólio. Toda a lógica de saldo opera com **créditos virtuais fictícios (R$)** armazenados localmente em memória. A plataforma não processa transações financeiras reais, dinheiro real, recargas ou mecânicas de apostas monetárias.

---

## 🛠️ Tecnologias Utilizadas
A arquitetura do ecossistema foi construída utilizando tecnologias modernas e limpas (Vanilla Stack) para garantir performance absoluta:

* **Backend:** Node.js, Express.js (Estrutura de API e rotas de autenticação simuladas).
* **Frontend:** HTML5, CSS3 (Design customizado baseado em variáveis modernas de estilo).
* **Lógica & Persistência:** JavaScript Vanilla (ES6+) e Web Storage API (`localStorage`).
* **UI/UX:** Capas de alta definição integradas via URLs dinâmicas com estética Neon/Cyberpunk.

## 🤖 Uso de Inteligência Artificial (ChatGPT/Gemini)
Este projeto foi desenvolvido com o suporte estratégico de ferramentas de Inteligência Artificial de forma iterativa. O modelo atuou no papel de um **Engenheiro de Software Full-Stack Sênior e Especialista em UI/UX**, sendo utilizado para:

1. **Engenharia de Prompt para o Google AI Studio:** Criação de diretrizes de contexto para blindar o chat contra falsos-positivos de segurança, permitindo o desenvolvimento limpo do código.
2. **Arquitetura Visual (CSS):** Estruturação do sistema de design *Cyber-Dark*, configuração das variáveis de cores neon (`#00ff88`, `#fbbf24`) e criação de animações fluidas para os modais de Login e Cadastro.
3. **Refatoração do Catálogo:** Varredura completa para remoção de jogos casuais antigos (como Asteroids e Runner) e implementação de uma grade focada em mecânicas modernas (Mines e Crash), além do reaproveitamento do código JavaScript básico original.
4. **Lógica de Interceptação (Gatekeeper):** Desenvolvimento da lógica de controle de acesso que impede usuários deslogados de interagirem com os mini-games, forçando a abertura dos formulários sem recarregar a página.
